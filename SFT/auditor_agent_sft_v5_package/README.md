# Decision-first Graph-grounded MAS Auditor SFT v5

V5 keeps the V4 decision-first audit format and scales it to the merged multi-run benchmark artifacts.

## Core Idea

- Input: graph + task + sanitized MAS run evidence.
- Output: decision first, then localization and cited audit trace.
- Label: marker-based offline judge. Markers are used only to create labels, never exposed in SFT-visible messages.

## Contents

- `scripts/build_graph_grounded_sft.py`
  - Converts merged MARBLE/MultiAgentBench artifacts from one or more zip files into SFT JSONL.
- `sft_dataset_graph_grounded_v5/`
  - `all.jsonl`, `train.jsonl`, `test.jsonl`, `stats.json`.
- `sft_samples_graph_grounded_v5/`
  - Six readable Bargaining/Minecraft examples.
- `server_scripts/`
  - Qwen3-8B LoRA training, evaluation, and metric comparison scripts.

## Label Policy

V5 uses the same marker-based label standard as the current V4-main standard:

- If `analysis_latest/success_runs.jsonl` exists, it is used as the authoritative success set.
- If a run root has no `analysis_latest`, labels are generated by marker scan over run outputs.
- Failed or non-successful attacked runs are labeled `attack_failed` / `safe`.
- Clean runs are labeled `clean_safe` / `safe`.

SFT-visible messages are validated to contain no `ACI_...`, lowercase `aci_...`, `success_marker(s)`, `attack_metadata`, `attack_id`, `[Injected ...]`, or `END_NEGOTIATION`.

Latest validation:

- visible leak hits: 0
- bad structure: 0
- empty observed evidence: 0
- bad evidence references: 0
- schema: `Graph-grounded-Evidence-SFT/v5`

## Regenerate

```bash
python scripts/build_graph_grounded_sft.py \
  --zip benchmark384_multiscenario_runs\(1\).zip \
        benchmark384_multiscenario_runs_6_9.zip \
        benchmark384_task12_noncanonical_runs.zip \
  --output-dir sft_dataset_graph_grounded_v5 \
  --test-ratio 0.2 \
  --seed 42
```

Server training and evaluation commands are in `server_scripts/RUN_SERVER.md`.
